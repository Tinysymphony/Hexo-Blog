AMD：Asynchronous Module Definition （异步模块定义）
CMD：Common Module Definition （通用模块定义）

说白了只是两个规范，有两个著名的JS分别实现了他们——RequireJS & SeaJS, 虽然都是老物了，下面结合代码做点笔记。

因为快要托福二战了，尽量用英文写博客~…


AMD

Why AMD — A detailed explanation of why we should use AMD to standardize modules

While IIFE cannot create manageable modules or have clear declaration of dependencies, CommonJS would be a better choice.

But when it commes to the browser environment, CommonJS may not be an optimal method because we have to translate the CJS-format modules to normal js files that browser can understand.

AMD shows its advantages on asynchronous loading and dependency management.

RequireJS

Just add this to your HTML, the index.js is the entry about dependency declaration.

In index.js, use require.config to inform the browser of all modules needed and all these will be loaded in an asynchronous way.

CMD

Common Module Definition
Sea.js is in CMD format and the most difference between AMD and CMD is that CMD will tend to load modules when they’re used, prefer a lazy-load.
